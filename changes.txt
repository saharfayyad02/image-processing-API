Summary of Test Files and Changes

spec/endpoint.spec.ts (API / Integration Tests)

Tests the HTTP endpoints using SuperTest.

Makes actual HTTP requests to the Express app, validating the full request/response cycle.

Covers:
✅ Health check endpoint (/health)
✅ Image resize API (/api/images)
✅ Legacy resize endpoint (/resize)
✅ Success scenarios (valid inputs)
✅ Error scenarios (invalid inputs, missing parameters)
✅ Edge cases (large numbers, decimal inputs, whitespace, concurrent requests)
✅ HTTP method validation
✅ 404 and 400 handling

Change in index.ts:

Fixed /resize endpoint to actually call resizeImage() and return the image buffer.
Returns 404 if the source image does not exist.
Sets correct Content-Type: image/jpeg header.
Validates query parameters and numeric dimensions (width, height) and returns 400 for invalid inputs.

Covers:
✅ resizeImage() function from imageService.ts

✅ Integration between core functions (logic pipeline)
